The first major hurdle you encounter when learning Linux is actually getting Linux installed on your computer. Installing a completely foreign operating system is daunting, and you are not willing to risk breaking your current Windows installation. Fortunately, we can use Cygwin, VirtualBox, and Vagrant to learn Linux without touching our current Windows installation.

In this article we'll create a virtual machine that has Linux already installed. Then you can play around with Linux in the virtual machine, and if you break something, you can spawn a new virtual machine with a fresh Linux install.

Cygwin

The first piece of software we'll need is Cygwin. Cygwin provides us with a Linux-like command line environment within Windows.

Cygwin Intro

The first thing you need to understand about Cygwin is that Cygwin is NOT a single program. Cygwin is an environment that contains many programs and packages. You can add or remove programs and packages from your Cygwin environment using the Cygwin Setup program. The "setup.exe" file that you download from the Cygwin website is the Cygwin Setup program.

Download Cygwin Setup

Please use the button below to download Cygwin Setup now.

Download Cygwin

After you've downloaded the Cygwin Setup program, move the "setup.exe" file to a convenient permanent location. Cygwin Setup will cache downloaded packages and package metadata in the same directory as the "setup.exe" file. Typically, you keep this file in the same place to take advantage of the caching feature. I usually place my "setup.exe" file in C:\cygwin-setup

Run Cygwin Setup

Now we're ready to run Cygwin Setup and install our first Cygwin packages. Double click the "setup.exe" file to run Cygwin Setup.

1. Welcome Screen

This welcome screen has some good information. Give it a read, then click Next >.

2. Choose Package Source

Choose "Install from Internet".

Click Next >.

3. Choose Cygwin Root Directory

Select the root directory where your Cygwin environment will be installed. I usually choose C:\cygwin for my Cygwin root directory.

Click Next >.

4. Choose Package Cache Directory

Choose where you want Cygwin Setup to store its cached package files. I usually store cached package files right next to "setup.exe" in C:\cygwin-setup.

Click Next >.

5. Configure Network Settings

Configure network settings appropriate for your Internet connection. I find that "Direct Connection" works well for me.

Click Next >.

6. Choose a Mirror

The official Cygwin package repository is available from multiple servers throughout the world. They all have the same Cygwin packages available, but download times vary from server to server. I usually just pick one at random.

If you get slow download speeds, try a different mirror. You can check out the Cygwin mirrors list and select a mirror that is close to your geographic location.

Choose a mirror, then click Next >.

7. Downloading Package Metadata

Cygwin Setup downloads the latest package metadata from your selected mirror and updates its local cache.

Sit back and let the download finish.

8. Select Packages to Install

Now we're ready to browse for packages and select the packages we want to install.

But first, let's explore the Cygwin package browser a little. Right now, we're looking at all Cygwin packages, organized by category.

Click the View button twice to see the packages that are currently selected for installation. (Hover your mouse over the View button for an explanation of the different "views".)

Right now all of the packages under the "Base" category are selected for installation. (The Base category includes all packages required for a minimal Cygwin installation.)

In addition to the base packages, we want to install the packages required for Cygwin/X. (Cygwin/X is required for us to run GUI applications in our Linux virtual machine.) According to the Cygwin/X website, Cygwin/X requires the following packages:

  • xorg-server
  • xinit
  • openssh

Switch to "Full" view, and type "xorg-server" into the search box.

The first package seems to be what we want.

Click the Skip button to cycle through options until the "New" column shows the latest version of xorg-server.
