The first major hurdle you encounter when learning Linux is actually getting Linux installed on your computer. Installing a completely foreign operating system is daunting, and you are not willing to risk breaking your current Windows installation. Fortunately we can use Cygwin, VirtualBox, and Vagrant to learn Linux without touching our current Windows installation.

In this article we'll create a virtual machine that has Linux already installed. Then you can play around with Linux in the virtual machine, and if you break something you can spawn a new virtual machine with a fresh Linux install. Using a virtual machine is a great way to work with Linux without altering your current Windows setup.

Cygwin

The first piece of software we'll need is Cygwin. Cygwin provides us with a Linux-like command line environment within Windows.

Cygwin Intro

The first thing you need to understand about Cygwin is that Cygwin is not a single program. Cygwin is an environment that contains many programs and packages. You can add or remove programs and packages from your Cygwin environment using the Cygwin Setup program. The "setup.exe" file that you download from the Cygwin website is the Cygwin Setup program.

Download Cygwin Setup

Please use the button below to download Cygwin Setup now.

Download Cygwin

After you've downloaded the Cygwin Setup program, move the setup.exe file to a convenient permanent location. Cygwin Setup will cache downloaded packages and package metadata in the same directory as the setup.exe file. Typically, you keep setup.exe in the same place to take advantage of the caching feature. I usually place my setup.exe file in C:\cygwin-setup.

Run Cygwin Setup

Now we're ready to run Cygwin Setup and install our first Cygwin packages. Double click the setup.exe file to run Cygwin Setup.

1. Welcome Screen

This welcome screen has some good information. Give it a read, then click Next >.

2. Choose Package Source

Choose "Install from Internet".

Click Next >.

3. Choose Cygwin Root Directory

Select the root directory where your Cygwin environment will be installed. I usually choose C:\cygwin for my Cygwin root directory.

Click Next >.

4. Choose Package Cache Directory

Choose where you want Cygwin Setup to store its cached package files. I usually store cached package files right next to setup.exe in C:\cygwin-setup.

Click Next >.

5. Configure Network Settings

Configure network settings appropriate for your Internet connection. I find that "Direct Connection" works well for me.

Click Next >.

6. Choose a Mirror

The official Cygwin package repository is available from multiple servers throughout the world. They all have the same Cygwin packages available, but download times vary from server to server. I usually just pick one at random.

If you get slow download speeds, try a different mirror. You can check out the Cygwin mirrors list and select a mirror that is close to your geographic location.

Choose a mirror, then click Next >.

7. Downloading Package Metadata

Cygwin Setup downloads the latest package metadata from your selected mirror and updates its local cache.

Sit back and let the download finish.

8. Select Packages to Install

Now we're ready to browse for packages and select the packages we want to install.

But first, let's explore the Cygwin package browser a little. Right now we're looking at all Cygwin packages, organized by category.

Click the View button twice to switch to the "Pending" view. The "Pending" view shows the packages that are currently selected for installation. Hover your mouse over the View button for an explanation of the different views.

Right now all of the packages under the "Base" category are selected for installation. The Base category includes all packages required for a minimal Cygwin installation.

In addition to the base packages, we want to install the packages required for Cygwin/X. Cygwin/X is required for us to run GUI applications in our Linux virtual machine. According to the Cygwin/X website, Cygwin/X requires the following packages:

  • xorg-server
  • xinit
  • openssh

Switch to "Full" view, and type "xorg-server" in the search box.

The first package seems to be what we want.

Click the Skip button to cycle through options until the "New" column shows the latest version of xorg-server.

Switch back to the "Pending" view to confirm that xorg-server is selected for installation. Notice that the base packages are hidden since we have typed "xorg-server" into the search filter box.

Click Clear to clear the search filter and see the full list of all pending operations. This unhides the base packages that are selected for installation.

Return to the "Full" view and use the search filter to find xinit and openssh. Select xinit and openssh for installation. Return to the "Pending" view to review your pending operations.

Click Next > to execute the pending operations.

Some of the packages we've selected depend on other packages. Cygwin Setup computes the required dependencies and offers to install them for us. This is a good thing.

Check the "Select required pacakges" checkbox, and click Next >.

Cygwin Setup downloads and installs the selected packages.

Sit back and let it run.

Congratulations! You've installed a base Cygwin environment with Cygwin/X.

Click Finish to exit Cygwin Setup.

Any time you want to install, uninstall, update, or reinstall packages in your Cygwin envirnoment, run the Cygwin Setup program. Remember that clicking the Skip button cycles through the possible options for a package. It's a confusing user interface, but you get used to it eventually.

Cygwin Directory Structure

Cygwin Setup installs all packages into the Cygwin root directory that you select. The Cygwin root directory no different from any other directory on your computer. It just happens to be the directory that the Cygwin environment considers to be it's root directory. You can browse the Cygwin root directory in Windows Explorer to see the files that are installed.

The rest of this article will assume that C:\cygwin is the Cygwin root directory.

The Cygwin environment also has a home directory for each user on your system. If your user name is "jdoe", then your Cygwin home directory is at C:\cygwin\home\jdoe.

Cygwin also creates a C:\cygwin\cygdrive directory. The C:\cygwin\cygdrive directory contains shortcut links to each drive on your computer. For example, C:\cygwin\cygdrive\c\Users is a shortcut to C:\Users. If you have a D:\ drive, you can access it via C:\cygwin\cygdrive\d. The shortcut links in C:\cygwin\cygdrive allow you to access all files on your computer from within the Cygwin environment.
