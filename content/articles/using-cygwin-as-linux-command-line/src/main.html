Intro

The first hurdle you encounter when learning Linux is actually getting Linux installed on your computer. Installing a foreign operating system is daunting, and you are not willing to risk breaking your current Windows installation. Fortunately we can use a virtual machine to learn Linux without touching our current Windows installation.

In this tutorial we'll create a Linux virtual machine. Then you can experiment with Linux on the virtual machine, and if you break something you can spawn a new virtual machine with a fresh Linux install. This tutorial is not meant to teach Linux. It is meant to give you access to a working Linux installation, and thus enable you to follow along with other Linux tutorials.

To work with Linux virtual machines in Windows, we'll install the following programs:

Notepad++
A text editor (similar to Windows Notepad) that recognizes both Linux-style line endings and Windows-style line endings. We'll use it to view and edit text files we create in the Linux virtual machine.
VirtualBox
Provides a platform for running virtual machines. This is the software that enables your computer to host virtual machines.
Vagrant
A command-line tool for quickly creating virtual machines (uses VirtualBox internally). We'll use this to get our virtual machine up and running quickly.
Cygwin
A Linux-like command line for Windows. We'll use this to run Vagrant on our Windows machine, and to run commands on the Linux virtual machine.

Cygwin Intro

Cygwin is different from the other program's we've installed so far.

Let me explain.

Cygwin Explained

The first thing you need to understand about Cygwin is that it is not a single program. A Cygwin installation is a collection of many programs and files which work together to form the Cygwin environment. These programs and files are distributed in the form of "Cygwin packages".

There are hundreds of packages available for Cygwin. Typically, you will install packages on an as-needed basis, instead of installing all Cygwin packages at once.

The Cygwin Setup utility manages the packages in your Cygwin installation. You can run Cygwin Setup at any time to:

  • Add new packages to your Cygwin installation.
  • Update existing packages in your Cygwin installation.
  • Remove unneeded packages from your Cygwin installation.

Packages work together, and many packages depend on other packages in order to function. These other required packages are called "dependencies". Typically, you tell the Cygwin Setup utility which packages you would like to install, and Cygwin Setup checks the packages you've selected to make sure all their dependencies are installed. If any dependencies need to be installed, Cygwin Setup offers to install the dependencies. This process is called "dependency resolution".

Road Map

To set up our Cygwin installation, we'll begin by downloading the Cygwin Setup utility (it's a single executable file). We'll place the Cygwin Setup utility in a permanent location, so we know where to find it in the future when we want to update our Cygwin installation.

Next, we'll launch the Cygwin Setup utility.

Cygwin Setup will first download a list of available packages from the Internet. Then it will open a package browser, where we can search through the available packages and choose the packages we want to install. Once we finalize our choices, Cygwin Setup will check if the packages we selected require any dependencies. Then, Cygwin Setup will download and install our selected packages and their dependencies.

Remember, you can always run Cygwin Setup again to add packages to your Cygwin installation, or remove packages from your Cygwin installation.

Cygwin Directory Structure

The rest of this article will assume that C:\cygwin is the Cygwin root directory.

There are a few directories under C:\cygwin that I'd like to point out. Open C:\cygwin in Windows File Explorer.

User Home Directories

Cygwin Setup creates a home directory for each user on your computer. Home directories appear under C:\cygwin\home. If your user name is "jdoe", then your Cygwin home directory is C:\cygwin\home\jdoe.

cygdrive Directory

Cygwin Setup creates a C:\cygwin\cygdrive directory. C:\cygwin\cygdrive contains shortcut links to each drive on your computer. For example, C:\cygwin\cygdrive\c is a shortcut to C:\. If you have a D:\ drive, you can access it via C:\cygwin\cygdrive\d. These shortcut links allow you to access all files on your computer from within the Cygwin environment.

Virtual Machine Setup

Intro

To create our Linux virtual machine, we'll start by using Vagrant to generate a generic Vagrantfile. The Vagrantfile is a configuration file that tells Vagrant what kind of virtual machine we want. We'll create a generic Vagrantfile using the vagrant init command on the command line. Then we'll use Notepad++ to edit a few lines in the Vagrantfile to customize our virtual machine.

Once we have our Vagrantfile, we'll run vagrant up to create the virtual machine. Vagrant reads the configuration in the Vagrantfile, then uses VirtualBox to create the virtual machine for us. After the virtual machine is created, Vagrant turns on the virtual machine. This is just like pressing the power button to turn on a physical computer.

Once the virtual machine is up and running, we need to connect to it. We'll run vagrant ssh to begin an SSH session on the virtual machine. While we are in the SSH session, all commands we enter on the command line are sent straight to the virtual machine. Using SSH is basically like opening a terminal on the virtual machine; the only difference is that we're doing it remotely through SSH. When we're ready to quit the SSH session, we'll run exit to close the SSH connection and return to our local command line.

After we exit the SSH session, the virtual machine is still running. We'll run vagrant halt to shut down the virtual machine. This is just like pressing the power button to shut down a physical computer.

Walkthrough

We need to create a directory to put our Vagrantfile in. Let's create a virt\linux directory under our Cygwin home directory, and put our Vagrantfile there. Open your Cygwin home directory in Windows File Explorer. Use Windows File Explorer to create the virt\linux directory under your Cygwin home directory.

Now we need to run vagrant init in our newly-created virt\linux directory to generate a generic Vagrantfile. We'll use Cygwin Terminal to do this. Start Cygwin Terminal through Cygwin/X.

Right now the terminal is in our Cygwin home directory. If we run vagrant init right now, it will create the generic Vagrantfile in our home directory. We want to create the Vagrantfile in the virt\linux directory we just created, not in our home directory. So, we have to change directories to the virt\linux directory before we run vagrant init.

Run cd virt/linux to move to the virt\linux directory you created.

Hint: Windows uses "\" (backslash) as a directory separator. Cygwin and Linux use "/" (forward slash) as a directory separator.

Now the terminal is in the virt\linux directory, and we are ready to run vagrant init. Run vagrant init to generate a generic VagrantFile in the virt\linux directory. Open the virt\linux directory in Windows File Explorer to see the newly-created Vagrantfile.

Note: "command not found" error
If you get a "command not found" error when you run vagrant init, it is because the vagrant.exe executable is not on your command line search path. The vagrant.exe file is installed in /cygdrive/c/HashiCorp/Vagrant/bin.

Run PATH="${PATH}:/cygdrive/c/HashiCorp/Vagrant/bin" to add the directory to your command line search path.

Now it's time to edit the generic Vagrantfile to customize our virtual machine configuration.

First, we'll tell Vagrant that we want to use the "ubuntu/trusty32" box for our virtual machine template. Let me explain this a bit.

Vagrant creates a virtual machine by cloning a "box". A "box" is a Vagrant virtual machine image. The "ubuntu/trusty32" box is a virtual machine image with Ubuntu installed. Ubuntu is a common Linux operating system. "Trusty" is a codename for Ubuntu version 14.04. There are many boxes to choose from, and you can even make your own. For now, we'll start with the "ubuntu/trusty32" box.

To use the "ubuntu/trusty32" box, we need to edit the Vagrantfile. Open the Vagrantfile in Notepad++, and change the line that reads config.vm.box = "base" to config.vm.box = "ubuntu/trusty32". This tells Vagrant to use the "ubuntu/trusty32" box as the template for our virtual machine.

Next, we need to tell Vagrant to allow X11 forwarding over SSH. X11 forwarding allows us to launch GUI applications during our SSH session. To do this, use Notepad++ to add a new line after config.vm.box = "ubuntu/trusty32". Type config.ssh.forward_x11 = true on the new line.

Save the edited Vagrantfile.

Now it's time to create the virtual machine. Go to the Cygwin Terminal and run vagrant up.

Vagrant reads the Vagrantfile, and gets to work downloading and cloning the "ubuntu/trusty32" box. After the box is downloaded and cloned, Vagrant starts up the newly-created virtual machine.

Now that the virtual machine has been created and started up, we can connect to it via SSH. Run vagrant ssh to start a new SSH session on the virtual machine.

Now we are in an SSH session on the virtual machine. Any commands we type are sent directly to the virtual machine, and run on the virtual machine.

Let's check the operating system version on the virtual machine. To do this, run lsb_release -a.

The command runs on the virtual machine, and tells us that the operating system is Ubuntu 14.04.

Now, let's shut down the virtual machine. To shut down the virtual machine, we need to end our SSH session first.

Run exit to end the SSH session, and return to your local computer's command line.

Now that we're back on our local computer, we can use Vagrant to shut down the virtual machine. Run vagrant halt to shut down the virtual machine.

Vagrant shuts down the virtual machine. Verify the status of the virtual machine by running vagrant status.

If we wanted to delete the virtal machine entirely, we could run vagrant destroy --force. Let's keep the virtual machine for now.

Virtual Machine Exploration

Intro

Let's explore our Linux virtual machine a bit.

We'll start this exploration by installing the following programs on the virtual machine:

Leafpad
A program for editing text files (similar to Windows Notepad).
PCManFM
A program for navigating files and folders (similar to Windows File Explorer).

We'll use Leafpad and PCManFM to create a new text file on the virtual machine. Then we'll copy the new file to a shared folder and view the file from Windows.

Leafpad and PCManFM are GUI applications. To use these GUI applications on our virtual machine, we'll need to run Cygwin Terminal through Cygwin/X.

Let's get started.

Walkthrough

Complete the following steps using the skills you've learned:

  1. Start Cygwin Terminal through Cygwin/X.
  2. Change to the virt\linux directory (cd virt/linux).
  3. Start the virtual machine (vagrant up).
  4. SSH into the virtual machine (vagrant ssh).

Now we're ready to install Leafpad and PCManFM on the virtual machine. To do this, we'll use Ubuntu's package manager, which is called "APT". APT is similar to the Cygwin Setup program we used before: you select packages you want to install, the package manager resolves dependencies, and the package manager downloads and installs the packages and dependencies. Let's ask APT to install Leafpad and PCManFM.

First, we should make sure APT has downloaded the most up-to-date package metadata. (Recall that Cygwin setup performed a similar step.) To download the latest package metadata, run sudo apt-get update.

Hint: Installing packages requires administrator privileges. sudo is used to run a command with administrator privileges.

Run sudo apt-get install leafpad pcmanfm to ask APT to install Leafpad and PCManFM. APT resolves the dependencies for Leafpad and PCManFM, and confirms the installation.

Enter y to confirm that you would like to install Leafpad and PCManFM, as well as the listed dependencies. APT gets to work downloading and installing the required packages.

Wait for it to finish.

Leafpad and PCManFM are now installed on the virtual machine. Now we can run Leafpad and PCManFM from the command line. Let's run PCManFM to take a look at the files and folders on our virtual machine.

Run pcmanfm.

The PCManFM GUI appears in a new window. To display the folders as a tree, click on the Places dropdown.

Select Directory Tree.

Now we can see all the files and directories on the virtual machine. Browse around the files to convince yourself that these are not the files on your Windows computer.

Let's use Leafpad to make a new text file on our virtual machine. Navigate back to /home/vagrant in PCManFM, and right click to bring up a context menu.

Select Create New... > Empty File to create a new empty file.

Name the new file test-file.

Click to create the new file.

Let's edit test-file in Leafpad. Right click on test-file to bring up a context menu.

Select Leafpad from the context menu.

A new Leafpad window opens our empty test-file. Type something silly in Leafpad. I typed "Hello Windows! My name is Linux!".

Let's save the edited test-file. Click File > Save.

Let's share our new test-file with Windows. We can do this by copying test-file into a shared directory. Conveniently, Vagrant has already created a shared directory for us.

By default, Vagrant shares the directory that our Vagrantfile is in with the virtual machine. The shared directory is accessed at /vagrant on the virtual machine. Navigate to /vagrant in PCManFM. You can see our Vagrantfile there. This is the same Vagrantfile that is on our Windows host in the virt/linux directory, under our Cygwin home directory.

Use PCManFM to copy and paste our test-file from /home/vagrant to /vagrant. Open the virt/linux directory in Windows File Explorer to confirm that the new test-file is there. Now open the test-file in Notepad++ to see the message we wrote on the Linux virtual machine.

Conclusion

The skills you've learned in this article enable you to interact with Linux on Windows. No Linux tutorial is out of your reach now. You can run commands on your Linux virtual machine with reckless abandon. If something breaks, run vagrant destroy --force to destroy the virtual machine, then run vagrant up to create a fresh virtual machine.

Keep in mind, the files under your shared directory are accessible from the Linux virtual machine, so they will be wiped out if you run something like rm -r /vagrant to delete the /vagrant directory. If you do run destructive commands like that, only the files and directories under the shared directory will be wiped out. Be mindful of this, and watch where you place your Vagrantfiles. Any subdirectory below the Vagrantfile can be wiped out by running rm -r /vagrant on the virtual machine.
